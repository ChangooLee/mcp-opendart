
# 📌 사용자 요청사항 정리

## 1. 목적
- MCP(mcp_opendart) 도구들의 `description`을 개선한다.
- 목표: LLM이 기업 공시 데이터에서 숨은 의도를 파악하고 리스크를 감지할 수 있도록 지원하는 데 최적화한다.
- 투자 관점 목적이 아닌 기업의 리스크를 분석하는데 주요 목적이 있다.

## 2. 개선 대상
- 각 도구의 `description` 필드만 개선한다.

## 3. 개선 방법 - 주어진 코드의 전체 도구를 분석한다.
- 요청받은 도구를 분석한다.
- 기존 코드에 있는 `description`은 참조한다.
- 반드시 코드의 함수 부분에 참조 : **OpenDART 공식 사이트의 API 상세 페이지**에 접속하여 실제 API 응답 구조를 직접 확인하고 작성한다.
- 실제 API 반환 필드와 그 의미를 **완전하게 분석**한 후 description을 작성한다.
- 주어진 API 응답 예시 및 레이아웃을 **반드시 참고**하여 정확성을 높인다.

## 4. 작성 형식
아래와 같은 구조로 작성한다 (필요시 유연하게 조정 가능):

- 【핵심 제공 데이터】
- 【연계 분석 도구】(※ 실제 MCP 도구에 존재하는 것만)
- 【활용 시나리오】
- 【효과적 활용 방법】
- 【주의사항 및 팁】(선택사항)

## 5. 작성 시 주의사항
- 한국어로 작성 (중국어식 표현 금지)
- 확대 해석 금지 (과장 표현 금지)
- 투자 권유 관점이 아닌 **기업 리스크 분석 관점**으로 작성
- 줄바꿈을 적극 사용하여 사용자 가독성 확보
- OpenDART가 제공하지 않은 항목은 작성하지 않는다
- 예시 데이터가 있을 경우 반드시 의미를 반영하여 작성
- 설명은 **실제 데이터에 기반**해야 한다
- 분석 초점은 **숨은 의도 파악**, **리스크 탐지**에 맞춘다
- 연계 분석 도구는 실제 MCP 코드에 존재하는 도구만 연결한다
- 【활용 시나리오】, 【효과적 활용 방법】은 현실적이고 실천 가능한 수준으로 작성한다

# 🔵 상세 작성 기준

## 【핵심 제공 데이터】
- API 응답 필드를 정확히 분석하여 제공
- 각 필드가 숨은 의도나 리스크 감지에 어떻게 기여할 수 있는지 설명
- 인사이트를 카테고리화하여 구성

## 【연계 분석 도구】
- 코드 상 실제 존재하는 MCP 도구만 연결
- 연계 분석의 논리적 흐름을 고려

## 【활용 시나리오】
- 여러 도구를 조합하여 기업의 숨은 의도, 리스크를 심층적으로 분석하는 구체적 방법 제시

## 【효과적 활용 방법】
- LLM이 도구를 효율적으로 사용하는 방법과 데이터 해석 방법 구체적으로 제시
- 서로 다른 필드나 도구 결과를 교차 분석하는 방법 제시

## 【주의사항 및 팁】(선택사항)
- 주의해야 할 점이나 최적 사용법이 있을 경우 추가

# 📌 추가 요청 사항

- **확대 해석 금지**: "이 데이터로 ~까지 분석할 수 있을 것이다" 식의 과장된 가정 금지
- description=“””내용”””, 형태의 결과로 다음줄을 반영하여 코드에 넣을 수 있도록 답변
- 답변 내용에 “ 등이 들어가면 안됨

# 7. description 예시

### 예시 1

> description="상장법인 및 주요 비상장법인이 제출한 주요사항보고서 중, 무상증자 결정을 공시한 내역을 조회하는 도구입니다. 기존 주주에 대한 무상 신주 배정 계획을 신속히 파악할 수 있으며, 주가 희석 가능성, 주주가치 제고 전략 여부, 주식 유통물량 변동성을 평가하는 데 활용됩니다.
>
> 【핵심 제공 데이터】
> - 주요 재무상태표 항목: 자산총계, 부채총계, 자본총계, 유동자산, 비유동자산 등
> - 주요 손익계산서 항목: 매출액, 영업이익, 법인세차감전이익, 당기순이익 등
> - 연결재무제표(CFS)와 개별재무제표(OFS) 모두 제공
> - 당기 및 전기 데이터 비교 가능
>
> 【연계 분석 도구】
> - get_stock_total: 무상증자 이후 총 주식수 변동 확인
> - get_major_holder_changes: 무상증자 이후 주요 주주 지분율 변동 추적
> - get_single_acc: 무상증자에 따른 자본금 변동 여부 분석
> - get_disclosure_list: 추가 무상증자 또는 유상증자 계획 여부 확인
>
> 【활용 시나리오】
> - 무상증자 규모 및 배정비율을 통해 기존 주주가치 희석 여부 평가
> - 신주배정 기준일 기준 권리락 발생 가능성 분석
> - 무상증자 이후 주식 수 급증에 따른 유통물량 변화 예측
> - get_stock_total, get_single_acc를 통해 자본구조 변동 추적
> - get_major_holder_changes를 통해 지배구조 변동성 사전 탐지
>
> 【효과적 활용 방법】
> - 신주발행수량(new_stock_co)와 기존 총발행주식수를 비교하여 희석율 계산
> - 1주당 배정주식수(per_stk_stock)를 활용하여 주가 조정 예상
> - 배정대상(target)을 검토하여 향후 지분변동 리스크 사전 점검
> - 무상증자 결정 이후 추가 공시(get_disclosure_list) 병행 모니터링"

### 예시 2

> description="기업의 연결재무제표를 통해 그룹 전체의 재무 건전성과 경영성과를 종합적으로 분석할 수 있는 핵심 도구입니다. 기업집단의 실질적 경제력과 잠재적 위험요소를 파악하는 데 필수적입니다.
>
> 【핵심 분석 요소】
> - 연결 매출액/이익: 그룹 전체의 실질적 사업 규모와 수익성 파악
> - 부채비율: 연결기준 재무 안정성과 차입 의존도 평가
> - 자본구조: 그룹 전체의 자본 건전성과 자본잠식 위험 분석
> - 시계열 변화: 3개년 재무지표 추이로 성장성과 지속가능성 평가
> - 내부거래 제거: 계열사 간 거래를 제외한 순수 외부 경영성과 확인
>
> 【지배구조 분석 활용법】
> - 연결/개별 비교: 자회사 의존도와 수익 기여도 파악(연결-개별 매출액 차이)
> - 위험요소 평가: 지배회사 외 종속회사 문제가 전체 그룹에 미치는 영향 분석
> - 차입 비교: 지배회사와 그룹 전체의 부채비율 차이로 숨겨진 부채 파악
> - 수익구조 분석: 지배회사와 종속회사 간 수익성 차이 및 그룹 내 역할 분석
>
> 【연계 분석 시나리오】
> - 기업 개황 파악: get_corporation_info로 기업 기본정보 및 결산월 확인
> - 공시 일정 확인: get_disclosure_list로 최근 재무제표 공시 여부 확인
> - 상세 계정 분석: get_single_acc로 전체 계정과목 세부내역 확인
> - 산업 비교 분석: 동종업계 여러 기업의 코드를 검색해 개별 get_single_acnt로 비교
> - 경영자 위험요소 점검: get_executive_trading으로 대표이사의 주식거래 분석
> - 재무비율 계산: get_single_index로 수익성/안정성/성장성/활동성 지표 확인
>
> 【분석 진행 단계】
> 1. 기본정보 확인: 연결대상 종속회사 수, 주요 계열사 현황 파악
> 2. 재무상태표 분석: 자산/부채/자본 구조와 전년 대비 변화 분석
> 3. 손익계산서 분석: 매출, 영업이익, 당기순이익의 추이와 변동성 평가
> 4. 현금흐름 분석: 영업/투자/재무활동 현금흐름의 건전성 점검
> 5. 종합 평가: 그룹 전체의 재무 건전성, 수익성, 성장성 종합 판단
>
> 【주의사항 및 팁】
> - 연결재무제표는 내부거래가 제거된 금액으로 그룹 외부와의 거래만 반영
> - 종속회사 변동(인수/매각)이 있는 경우 전년 대비 단순 비교는 왜곡될 수 있음
> - 재무제표 주석사항에서 연결대상 종속회사 목록 및 지분율 확인 필요
> - 연결기준 부채비율과 개별기준 부채비율의 큰 차이는 종속회사의 과도한 차입 의심"
